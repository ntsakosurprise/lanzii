# Issues

We welcome bug reports, feature requests, and general feedback.  
To help us resolve issues efficiently, please follow the guidelines below.

---

## **1. Search Before Reporting**

Before creating a new issue, please:

- Check the [existing issues](https://github.com/ntsakosurprise/anzii/issues) to see if your problem has already been reported
- Search the documentation for related topics

---

## **2. What Types of Issues Can Be Reported**

- **Bug reports:** Unexpected behavior, errors, or crashes
- **Feature requests:** Suggestions for new features or enhancements
- **Documentation improvements:** Typos, missing guides, or unclear explanations
- **Performance issues:** Bottlenecks or slow behavior

---

## **3. How to Create a Good Issue**

When creating an issue, please include:

1. **Title:** Concise and descriptive
2. **Description:** Explain the problem or suggestion clearly
3. **Steps to Reproduce (for bugs):**
   - Environment (OS, Node version, browser, etc.)
   - Exact steps to reproduce
   - Expected behavior
   - Actual behavior
4. **Screenshots or Logs:** If relevant, attach images or error messages
5. **Additional context:** Anything else that might help

---

## **4. Issue Templates**

If available, please use the GitHub issue templates:

- [Bug Report Template](https://github.com/your-repo/issues/new?assignees=&labels=bug&template=bug_report.md)
- [Feature Request Template](https://github.com/your-repo/issues/new?assignees=&labels=enhancement&template=feature_request.md)

Using templates ensures your issue has all the required information.

---

## **5. Code of Conduct**

All contributors and reporters must follow the [Code of Conduct](CODE_OF_CONDUCT.md).

---

## **6. Closing Issues**

- Issues that are invalid, duplicates, or resolved may be closed by maintainers
- If your issue is closed prematurely, feel free to comment with additional information
